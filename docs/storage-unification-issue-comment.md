ローカル保存版と Firestore 版は、UI と業務ロジックを共通化し、保存処理のみをストレージ層へ分離する構成に整理しました。

記録先:
- `docs/storage-unification.md`

今回の整理内容:
- `js/platform` に保存基盤を集約
- `js/storage/durable-items` と `js/storage/pc-items` で保存方式ごとの差分を分離
- UI 側は保存方式を直接意識せず、共通の保存窓口を利用

補足:
- 現在の切替はビルド時環境変数ではなく、ローカルモードフラグによるランタイム切替です
- 将来的に別ビルド公開が必要な場合も、現行の分離構成を土台に拡張可能です
